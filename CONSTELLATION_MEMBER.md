# Orchestra membership — janusplot

> **This project (janusplot) is a MEMBER of the Orchestra** (one coordination
> structure; reconciled 2026-06-03). The leader-node is **ORCHESTRA_dev**
> (governance, roster, contracts, TACI, publication); the technical inference hub
> is **flexyBayes** (the dependency-DAG sink). This file is janusplot's
> back-pointer to that charter and a map of my siblings.

- **My role:** the **diagnostic-visualisation member** — asymmetric, directional,
  non-linear pairwise-association matrices via penalised GAM smooths, with analytic
  derivatives + simultaneous CIs and an objective 24-category shape classifier
  (the "what shape and which direction is this relationship" lens a scalar
  correlation matrix hides). Candidate diagnostic-visualisation hub (P4).
- **Contracts I own / honour:** I emit **no manifest** (a diagnostic member). I
  consume tabular data; I produce a figure + a shape taxonomy. The asymmetry index
  is a non-linear causal-direction **screen**, never a causal verdict.
- **My edge:** diagnostic — janusplot reads any member's data/residuals and types
  the association shapes (dose-response, compensation, crossover). Complements
  kernR's kernel dependence statistic with a *visual, directional, shape-typed*
  view. No hard dependency on another member.
- **What binds me (charter invariants):** *single-responsibility* (association
  diagnostics + shape typing, not a causal test or a decision); *honest
  per-cell degradation* (`indeterminate`/`flat` categories + `k_check`
  basis-deficiency flags, not a fabricated shape); *leader-directed adoption*.
- **Governance:** janusplot is a **Max-owned personal package** (MIT-track);
  the AAGI-AUS canon does not apply.
